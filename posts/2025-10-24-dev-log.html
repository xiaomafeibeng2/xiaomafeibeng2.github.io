打造可复用的色彩系统

当产品规模扩张时，色彩管理常常成为痛点。高饱和的强调色、无法复用的灰度、品牌色不统一，都会让体验显得杂乱。这篇记录整理了我在最近项目中设计色彩系统的一些经验。

色彩结构

我将色彩分成三个层级： Brand（品牌识别）、Semantic（语义反馈）和 Neutral（中性色）。每个层级都有明确的使用场景，避免在组件中随意取色。

设计与开发的接口

在设计稿中，我会标注 token 名称而不是 HEX 值；在代码中，则通过 CSS 变量或设计系统主题文件引用。例如： 

:root {
  --color-brand: #1f65ff;
  --color-brand-soft: rgba(31, 101, 255, 0.12);
  --color-success: #22c55e;
  --color-danger: #ef4444;
}

这样做的好处是，当品牌色有调整时，只需要在一个地方修改变量即可，全站同步。

测试与迭代

除了常规的深浅对比度检查，我还会制作一个“状态板”，列出所有组件在不同状态下的表现，以便快速识别不协调的组合。
